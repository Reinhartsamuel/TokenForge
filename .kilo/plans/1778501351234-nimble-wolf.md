# TokenForge Landing Page Plan

## Overview
Create a separate public-facing marketing landing page for TokenForge to convert visitors (developers, issuers, investors) into users. This will be a standalone Next.js app within the existing `dashboard/` project as a new route group, keeping deployment simple while maintaining separation from the issuer dashboard app.

## Architecture Decision
- **Location:** `dashboard/app/(landing)/` - Route group within existing Next.js app
- **Rationale:** Single deployment, shared components/ui library, separate layout from dashboard
- **Dashboard stays at:** `dashboard/app/(dashboard)/` - Move existing app into route group

## Design System

### Visual Direction (from design-taste + brand-design)
- **Style:** Dark developer tooling aesthetic with flat design principles
- **Colors:** Use Solana brand palette from pitch deck:
  - Primary accent: `#14F195` (Solana green)
  - Secondary accent: `#9945FF` (Solana purple)
  - Dark background: `#0F0F23`
  - Card surface: `#1A1A3E`
  - Text: `#FFFFFF`, `#8B8BA7` (muted)
- **Typography:** Inter (sans) + JetBrains Mono (code) - already in dashboard
- **Density:** Comfortable for marketing, asymmetric hero + symmetric data sections

### Anti-Patterns to Avoid
- No emojis as icons (use Lucide SVG icons)
- No `transition-all` (use specific properties)
- No pure black `#000` in product UI
- No layout shift on hover states
- Ensure 4.5:1 contrast ratio minimum

## Technical Considerations

### Routing Strategy
- Landing page at root: `/`
- Dashboard accessible at: `/dashboard` or `/app`
- Use Next.js route groups to separate layouts cleanly

### Shared Components
- Reuse existing UI components from `@/components/ui/`
- Share color tokens from `globals.css`
- Use same Lucide icon library

### Performance
- Static generation where possible (landing page content doesn't change often)
- Optimize images and SVGs
- Minimal JavaScript for landing page (mostly static HTML)

### SEO & Meta
- Proper title: "TokenForge — The Metaplex of RWA Tokens"
- Description: "Open-source SDK and issuer dashboard for Solana Security Token Standard"
- Open Graph tags for social sharing
- Favicon from existing dashboard

## Success Metrics
- Clear value proposition above the fold
- Developer can install SDK within 2 clicks
- Investor can understand market opportunity
- Compliance team sees no-code dashboard benefits
- Mobile experience is fully functional
