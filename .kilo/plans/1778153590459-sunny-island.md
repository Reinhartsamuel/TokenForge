# TokenForge Hackathon Plan — Demo Flow & Submission

> **Hackathon submission deadline: 3 days**
> **Positioning:** TokenForge is the first issuer platform that makes regulated security tokens work on Solana using the Halborn/Hoodies/Securitize SSTS standard.

## Context

All Phase 1 issues are resolved. Token creation via backend keypair signer works end-to-end on devnet. The MetaMask Solana Snap has known incompatibilities with our transaction format — we've migrated to a backend keypair signer approach which works perfectly.

## Resolved Issues

| Issue | Root Cause | Fix | Status |
|-------|-----------|-----|--------|
| `canonicalIxToWeb3` role detection | `@solana/kit` v2 uses numeric role field (0,1,3,5) instead of boolean flags | Added role-based detection + extraction to `dashboard/lib/sdk/l1/transactions.ts` | ✅ Done |
| Address string → bytes mismatch | Base58 address strings exceed 32-byte seed limit for PDA derivation | Use `bs58.decode(addr)` in derive.ts for all PDA seeds | ✅ Done |
| Wrong PDA seed prefix | Used `"mint_authority"` (underscore) instead of `"mint.authority"` (dot) | Fixed to match on-chain `seeds::MINT_AUTHORITY = b"mint.authority"` | ✅ Done |
| Wrong PDA seed composition | Used only creator bytes, missing mint bytes | Fixed to `[mint.authority, mint_bytes, creator_bytes]` matching on-chain | ✅ Done |
| Wrong freezeAuthority in args | Passed `mintAuthorityPda` instead of derived freeze authority PDA | Added `deriveFreezeAuthorityPda` with seeds `[mint.freeze_authority, mint_bytes]` | ✅ Done |
| Signer type mismatch | Passed web3.js Keypair/PublicKey where @solana/kit expects Address strings | Use `createNoopSigner(address)` for instruction builder inputs | ✅ Done |
| MetaMask Snap incompatibility | MetaMask's Solana Snap disables confirm button on simulation | Migrated to backend keypair signer via `/api/test-create-token` endpoint | ✅ Done |
| `confirmTransaction` API mismatch | web3.js v1.98.x expects object-based confirmation strategy | Use `{ signature, blockhash, lastValidBlockHeight: BigInt() }` | ✅ Done |
| `partialSign` return type | web3.js v1.98.x `partialSign` returns void (mutates in place) | Remove assignment: `tx.partialSign(keypair)` instead of `tx = tx.partialSign(keypair)` | ✅ Done |

### Day 1: Mint Tokens (TODAY)

**Goal:** Mint form working via backend keypair signer, wired to dashboard.

#### 1.1 Create `/api/mint-tokens` endpoint

**File:** `dashboard/app/api/mint-tokens/route.ts` (new)

Server-side endpoint that:
- Reads mint address, destination, amount from request body
- Uses TEST_WALLET_KEYPAIR from .env for signing
- Derives MintAuthority PDA + VerificationConfig PDA (discriminator 6)
- Derives or creates destination ATA (auto-create if missing)
- Builds canonical Mint instruction
- Converts via `canonicalIxToWeb3`, signs, sends
- Returns: success/error, signature, mint, destination ATA, explorer URL

**Workflow:**
```
1. Parse request: { mintAddress, destination, amount }
2. Derive mintAuthorityPda = deriveMintAuthorityPda(mint, creator)
3. Derive verificationConfigPda = deriveVerificationConfigPda(mint, 6)
4. Get/create destination ATA via getOrCreateAssociatedTokenAccount
5. Build Mint instruction via getMintInstruction
6. Convert to web3.js via canonicalIxToWeb3
7. Sign with TEST_WALLET_KEYPAIR, send, confirm
8. Return result
```
